E-Commerce

#eccomerce #php #React JS

Mon projet récent en tant que développeur consiste en la création d'un portfolio interactif intégrant une boutique de livres fictifs. En utilisant React.js pour le front-end et PHP pour le back-end, j'ai développé une interface dynamique permettant aux utilisateurs de parcourir, acheter, et effectuer des paiements sécurisés via l'intégration de Stripe . Pour faciliter la gestion du contenu, j'ai ajouté une interface administrateur qui permet aux administrateurs d'ajouter, mettre à jour, ou supprimer des livres en vente. La communication entre le front-end et le back-end est assurée par Axios , garantissant des échanges de données fluides.

Api - JS

#eccomerce #php #React JS

En utilisant JavaScript, j'ai développé une barre de recherche qui permet aux utilisateurs de rechercher des recettes dans la base de données de TheMealDB. Lorsqu'une recherche est effectuée, les détails de la recette correspondante s'affichent. De plus, j'ai intégré une fonctionnalité permettant d'obtenir jusqu'à six recettes aléatoires lorsqu'un utilisateur clique sur le bouton dédié. Chaque résultat de recherche ou recette aléatoire est présenté sous forme de carte, comprenant une image, le nom de la recette, et un lien pour afficher plus d'informations.

Spring Security

#java #spring #security

Dans mon récent projet, j'ai développé une application en utilisant Spring Boot avec une emphase particulière sur la sécurité. En intégrant le starter Spring Security , j'ai configuré l'authentification dans Spring pour garantir un accès sécurisé à l'application. Pour renforcer la sécurité, j'ai mis en œuvre la génération d'un token JWT (JSON Web Token), puis j'ai géré le traitement du token à l'aide d'un filtre dédié. Ce filtre assure la validation et l'authentification des utilisateurs à chaque requête, offrant ainsi une protection. De plus, j'ai sécurisé différentes ressources en attribuant des rôles aux utilisateurs , assurant ainsi un contrôle d'accès approprié.

POO - Java

#java #POO #class

Le jeu Tic Tac Toe est conçu avec une approche de conception orientée objet (POO) en Java. Les classes clés telles que Game, Cell, et Player encapsulent des fonctionnalités spécifiques. La classe Game gère la logique du jeu avec des méthodes comme move(), isValidMove(), et checkPlayerWin(). La classe Player définit les caractéristiques individuelles des joueurs. Cette structure favorise une organisation claire et modulaire du code, améliorant la lisibilité et la maintenance du projet.
